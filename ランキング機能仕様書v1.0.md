## 1. 概要

### 1.1. 目的
ステージごとのグローバルランキング (TOP 100) を表示する機能を追加する。α/β 段階でのバイラル強化と、ユーザー継続プレイのモチベーション創出が目的。

### 1.2. ゴール / 非ゴール

| ゴール | 非ゴール (やらない) |
|---|---|
| ステージごとの TOP 100 を表示 | アカウント / フレンド機能 |
| 自分の順位ハイライト表示 | 週間 / 月間ランキング |
| クリア時に自動送信 (設定で OFF 可) | チャット / コメント |
| ニックネーム or PlayerID で参加 | ゲーム内チート (JS改造) 防止 |
| **外部からの DB 直接改竄を完全ブロック** | スコア再計算によるチート検出 |

### 1.3. 用語
- **PlayerID**: 初回起動時に自動発行される 6 桁英数字 (例: `Z4QLN6`)。ゲーム内で不変。
- **DisplayName**: プロフィールで設定したニックネーム (最大 16 文字) または PlayerID そのもの。`Save.getPlayerDisplayName()` が返す値をそのまま使う。
- **IconID**: プロフィールで選んだアイコン ID (`sprite/avatars/manifest.json` の id)。未選択時は `null`。

- **Cloud Functions 不使用** (α/β スコープでは不要)

### 2.1. 脅威モデルと防御

| 攻撃 | 防御 |
|---|---|
| 外部から `curl`/`Postman` で `/rankings` に任意データ注入 | App Check で拒否 |
| 漏洩した API キーでの API 直叩き | App Check で拒否 |
| MITM (Charles / Proxyman) でのリプレイ | App Check トークン短寿命 (1h) で不可 |
| 既存レコードの改竄 | Security Rules で `update` 禁止 |
| 既存レコードの削除 | Security Rules で `delete` 禁止 |
| 範囲外の値の書き込み (例: score=999999) | Security Rules でフィールドバリデーション |
| ゲーム内でのスコア水増し (JS 改造) | **α/β スコープでは許容**。チーターが現れたら後日 Cloud Functions 追加 |

## 4. Firestore データモデル (🤖 AI が設計・実装)

### 4.1. コレクション構造

```
/rankings/{stageId}/scores/{docId}
  - playerId:    string     6桁英数字 必須
  - displayName: string     最大 16 文字 必須 (空文字不可)
  - iconId:      string|null アイコン ID、未選択時は null
  - score:       number     0 〜 40000 必須
  - correct:     number     0 〜 20 必須 (正解問題数)
  - totalTimeMs: number     0 〜 1800000 必須 (合計回答時間、最大 30 分)
  - difficulty:  string     'EASY' | 'STRESS' | 'EXTREME' のいずれか
  - rank:        string     'SS'|'S'|'A'|'B'|'C'|'D'|'E'|'F' のいずれか
  - deathEnd:    boolean    即死終了フラグ
  - createdAt:   timestamp  serverTimestamp()
  - sessionId:   string     UUIDv4 (重複送信検知用)
  - appVersion:  string     `${CONFIG.VERSION}` (例: 'v0.1.0-alpha')
```

- `stageId` は `'stage-1'` 〜 `'stage-10'` (既存 CONFIG.STAGES に合わせる)
- `docId` は Firestore の autoId
- **個人情報 (メールアドレス、本名、位置情報など) は一切保存しない**

### 4.2. 複合インデックス
ランキング取得のため以下を作成:

```
Collection: /rankings/{stageId}/scores
Fields: score DESC, createdAt ASC
```

Firestore は 1 フィールドのみの orderBy はインデックス不要。今回は `score` 降順のみでも可 (同点時の tiebreak は `createdAt` 昇順 = 先にクリアした方が上)。

### 4.3. 容量試算
1 レコード ≈ 400 byte。DAU 10,000 × 10 ステージ = 100,000 レコード/日 でも 40MB/日。Spark プランの Firestore 無料枠 (1 GB ストレージ) で **25 日分保存可能**。

⚠️ 後日古いレコード自動削除 (30 日 TTL) を検討。α/β では不要。

## 5. Firestore Security Rules (🤖 AI が書く)

### 5.1. ファイル
プロジェクトルートに `firestore.rules` を作成する。

### 5.2. 内容

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // ---------- Ranking scores ----------
    match /rankings/{stageId}/scores/{docId} {

      // 誰でも読める (TOP 100 を返すため)
      allow read: if true;

      // create のみ許可
      allow create: if isValidRankingCreate(stageId);

      // update/delete は完全禁止 (コンソールから手動のみ)
      allow update, delete: if false;
    }

    // 上記以外は全面禁止 (明示的 allow が無い限り deny)
    match /{document=**} {
      allow read, write: if false;
    }
  }
}

function isValidRankingCreate(stageId) {
  let data = request.resource.data;
  let VALID_STAGES = ['stage-1','stage-2','stage-3','stage-4','stage-5','stage-6','stage-7','stage-8','stage-9','stage-10'];
  let VALID_DIFFICULTY = ['EASY','STRESS','EXTREME'];
  let VALID_RANK = ['SS','S','A','B','C','D','E','F'];

  return
    // 正しいステージパス配下であること
    stageId in VALID_STAGES

    // フィールド過不足なし
    && data.keys().hasOnly([
        'playerId','displayName','iconId','score','correct','totalTimeMs',
        'difficulty','rank','deathEnd','createdAt','sessionId','appVersion'
    ])
    && data.keys().hasAll([
        'playerId','displayName','iconId','score','correct','totalTimeMs',
        'difficulty','rank','deathEnd','createdAt','sessionId','appVersion'
    ])

    // 型と範囲
    && data.playerId    is string && data.playerId.matches('^[A-HJ-NP-Z2-9]{6}$')
    && data.displayName is string && data.displayName.size() >= 1 && data.displayName.size() <= 16
    && (data.iconId == null || (data.iconId is string && data.iconId.size() <= 64))
    && data.score is number       && data.score >= 0       && data.score <= 40000
    && data.correct is number     && data.correct >= 0     && data.correct <= 20
    && data.totalTimeMs is number && data.totalTimeMs >= 0 && data.totalTimeMs <= 1800000
    && data.difficulty in VALID_DIFFICULTY
    && data.rank in VALID_RANK
    && data.deathEnd is bool
    && data.createdAt == request.time                   // serverTimestamp 強制
    && data.sessionId is string && data.sessionId.matches('^[0-9a-f-]{36}$')
    && data.appVersion is string && data.appVersion.size() <= 32
    ;
}
```

## 7. `js/online/ranking.js` — 送信・取得クライアント (🤖 AI が実装)

### 7.1. ファイル
`js/online/ranking.js` を新規作成。

### 7.2. 公開 API
```js
window.Ranking = {
  submitScore(stageId, result) → Promise<{ok: boolean, reason?: string}>
  fetchTop(stageId, limit=100) → Promise<Array<RankRow>>
  fetchMyRank(stageId, playerId) → Promise<{rank: number, row: RankRow} | null>
  isEnabled() → boolean          // 設定 ON/OFF
  setEnabled(v: boolean)
};
```

### 7.3. スケルトン

```js
/* ============================================================
   ranking.js — オンラインランキング送信・取得
   ------------------------------------------------------------
   - Firebase は index.html 側で初期化済み (window.Firebase)
   - 送信は Save の getPlayerDisplayName() / getPlayerIcon() を使う
   - 設定で OFF にされている場合は何もしない
   ============================================================ */

(function () {
  const COLL_ROOT = 'rankings';

  // 設定キー (Save.data.settings.onlineRanking)
  function isEnabled() {
    const s = window.Save?.getSettings?.();
    return s?.onlineRanking !== false; // デフォルト true
  }

  function setEnabled(v) {
    window.Save?.updateSettings?.({ onlineRanking: !!v });
  }

  // セッション ID 発行 (1プレイ = 1 UUID)
  function newSessionId() {
    // crypto.randomUUID() が iOS Safari 15+ で使える
    if (crypto?.randomUUID) return crypto.randomUUID();
    // polyfill
    return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, c => {
      const r = Math.random() * 16 | 0;
      const v = c === 'x' ? r : (r & 0x3 | 0x8);
      return v.toString(16);
    });
  }

  async function submitScore(stageId, result) {
    if (!isEnabled()) return { ok: false, reason: 'disabled' };
    if (!window.Firebase?.db) return { ok: false, reason: 'firebase-not-ready' };

    const { addDoc, collection, serverTimestamp } =
      await import('https://www.gstatic.com/firebasejs/10.14.1/firebase-firestore.js');

    const playerId   = window.Save?.getPlayerId?.() || '';
    const dispName   = window.Save?.getPlayerDisplayName?.() || playerId;
    const iconId     = window.Save?.getPlayerIcon?.() || null;

    const payload = {
      playerId,
      displayName: dispName.slice(0, 16),
      iconId,
      score:       Math.max(0, Math.min(40000, result.score | 0)),
      correct:     Math.max(0, Math.min(20, result.correct | 0)),
      totalTimeMs: Math.max(0, Math.min(1800000, result.totalTimeMs | 0)),
      difficulty:  result.difficulty || 'EASY',
      rank:        result.rank || 'F',
      deathEnd:    !!result.deathEnd,
      sessionId:   newSessionId(),
      appVersion:  window.CONFIG?.VERSION || 'unknown',
      createdAt:   serverTimestamp(),
    };

    try {
      const ref = collection(window.Firebase.db, COLL_ROOT, stageId, 'scores');
      await addDoc(ref, payload);
      return { ok: true };
    } catch (e) {
      console.warn('[Ranking] submit failed:', e);
      return { ok: false, reason: e.code || 'unknown' };
    }
  }

  async function fetchTop(stageId, limit = 100) {
    if (!window.Firebase?.db) return [];
    const { collection, query, orderBy, limit: qLimit, getDocs } =
      await import('https://www.gstatic.com/firebasejs/10.14.1/firebase-firestore.js');

    const ref = collection(window.Firebase.db, COLL_ROOT, stageId, 'scores');
    const q = query(ref, orderBy('score', 'desc'), orderBy('createdAt', 'asc'), qLimit(limit));
    const snap = await getDocs(q);
    return snap.docs.map((d, i) => ({ rank: i + 1, id: d.id, ...d.data() }));
  }

  async function fetchMyRank(stageId, playerId) {
    // TOP 100 から自分を探す (TOP 100 外なら別クエリで自分のベストだけ取得)
    const top = await fetchTop(stageId, 100);
    const mine = top.find(r => r.playerId === playerId);
    if (mine) return { rank: mine.rank, row: mine };

    // TOP 外なら個別に自分のベストを取得
    if (!window.Firebase?.db) return null;
    const { collection, query, where, orderBy, limit, getDocs } =
      await import('https://www.gstatic.com/firebasejs/10.14.1/firebase-firestore.js');
    const ref = collection(window.Firebase.db, COLL_ROOT, stageId, 'scores');
    const q = query(ref, where('playerId', '==', playerId), orderBy('score', 'desc'), limit(1));
    const snap = await getDocs(q);
    if (snap.empty) return null;
    const row = { id: snap.docs[0].id, ...snap.docs[0].data(), rank: null };
    return { rank: null, row };
  }

  window.Ranking = { submitScore, fetchTop, fetchMyRank, isEnabled, setEnabled };
})();
```

⚠️ 動的 `import()` は `index.html` 側で Firebase SDK のインポートマップを有効にしておく必要がある。セクション 6.2 の `<script type="module">` で既にロード済みなので、二重ロードにならないよう **CDN 側が自動で同一モジュールをキャッシュ**する (HTTP 304)。

### 16.1. Global Rating (全ステージ横断ランキング)

**問題意識**:
現状の素点 (score) はどのステージでも最大 40,000 点。なので単純合計で並べると
Stage 1 を 10 回プレイした人と Stage 10 を 10 回プレイした人の「実力」が同じに
見えてしまう。Global ランキングを作るなら**ステージ難易度係数**で重み付けが必要。

**ユーザー要件 (2026-04 ヒアリング)**:
> 「Stage 1 の SS でも Stage 2 の A には勝てない」くらいの傾斜

**推奨重み**:

| Stage | DIFF_WEIGHT | 前段比 |
|---|---:|---:|
| 1  | 1.0  | — |
| 2  | 1.4  | ×1.4 |
| 3  | 2.0  | ×1.4 |
| 4  | 2.8  | ×1.4 |
| 5  | 4.0  | ×1.4 |
| 6  | 5.6  | ×1.4 |
| 7  | 8.0  | ×1.4 |
| 8  | 11.0 | ×1.4 |
| 9  | 16.0 | ×1.4 |
| 10 | 22.0 | ×1.4 |

**検算 (「Stage N-1 SS < Stage N A」ルール)**:
- Stage 1 SS = 40,000 × 1.0 = **40,000**
- Stage 2 A (85%目安 ≈ 34,000) × 1.4 = **47,600** ✓
- Stage 2 SS = 40,000 × 1.4 = **56,000**
- Stage 3 A (34,000) × 2.0 = **68,000** ✓
- Stage 9 SS = 40,000 × 16.0 = **640,000**
- Stage 10 A (34,000) × 22.0 = **748,000** ✓

**Global Rating = 各ステージベスト Weighted Score の合計**
全 10 ステージ全問正解 (SS) なら: 40,000 × (1+1.4+2+2.8+4+5.6+8+11+16+22) = **40,000 × 73.8 = 2,952,000 点**

**実装時の変更箇所**:
- `js/scoring.js` に `DIFF_WEIGHT = {1:1.0, 2:1.4, ...}` を追加
- `computeWeightedScore(rawScore, stageNo)` ヘルパー追加
- `js/save.js` に「各ステージの Weighted ベスト」を保持する領域追加
- `js/online/ranking.js` の `submitScore` 時に `weightedScore` フィールドも送信
- `js/screens/ranking.js` で Global Rating 画面を追加 (ステージ横断 TOP 100)
- Firestore に `/rankings/__global__/ratings/{playerId}` コレクション追加

**結果画面への表示は任意**:
- 既存の素点 (score) 表示はそのまま
- 下に小さく「難易度補正: ○○○ pt」を出すかどうかは UX 検討 (今は出さなくて可)

**注意**:
この重み付けはあくまで Global Rating の集計用。**各ステージ内の順位表示 (現在の percentileText) はステージ内のパーセンタイルを続投**する (既に `STAGE_RANK_PCT` で実装済み)。ユーザーが「Stage 1 の SS は上位 40%、Stage 10 の SS は上位 0.001%」と認識するのはこちら側。

## 付録 B: よくある質問

**Q. API キーがソースに入るけど安全?**  
A. Firebase の API キーはもともと公開前提の識別子。**App Check で「本物アプリからのリクエストか」を検証**するので、キーが漏れても他のクライアントから呼び出しに使えない。

**Q. Firebase の無料枠を超えたら?**  
A. Spark プランの Firestore 無料枠は「1 日 50,000 読み取り / 20,000 書き込み」。超えると **課金ではなくエラー応答に切り替わる**ので、いきなり請求が来る事故は起きない。Blaze プランに切り替えるまではゲーム側が「ランキング取得失敗」として動くだけ。

**Q. オフラインでプレイしたときの送信は?**  
A. Firebase SDK は**自動でキューに保持 → オンライン復帰時に送信**する (デフォルト動作)。

**Q. 複数端末で同じプレイヤーが遊ぶ?**  
A. PlayerID は `localStorage` 保存なので端末ごとに別。同じ人扱いしたい場合はアカウント機能が必要 (スコープ外)。

**Q. ニックネーム重複は?**  
A. 許容する。表示上は PlayerID が違うので識別可能。悪用防止目的の重複チェックはしない (NG ワードフィルタは将来検討)。
