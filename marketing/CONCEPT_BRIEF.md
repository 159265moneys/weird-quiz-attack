## 2. コンセプト核 (これを外したら死ぬ)

### 2-1. ジャンル感
**「クソゲー」は誇張ではなく企画コア**。プレイヤーに **「これクソゲーだろ wwww」と笑いながらシェアさせる** ことが最大目標。なので "上品で美しいクイズアプリ" にしない。**意図的に壊れて見せる**。

### 2-2. 3 軸の対比

| 軸 | 表 (まとも) | 裏 (崩壊) |
|---|---|---|
| 画面 | タイトル / メニュー / 結果画面 | 出題画面 |
| 感情 | 「ふつうのクイズアプリかな」 | 「は???? なんで画面回ってんの????」 |
| トーン | クール・整った見た目 | グリッチ・崩壊・煽り |

→ **このギャップが笑いを生む**。最初は普通の顔して油断させ、ステージが進むほど狂う。

### 2-3. 共感バズワード
ユーザーが「これ習ったっけ…」となる **忘れてそう系** を意図的に押し出す。  
**例: 維管束、班田収授法、まつり縫い、変ロ長調、勘合貿易**

App Store 説明文や告知画像に **「あなたは維管束を覚えていますか?」** みたいな煽り文を載せる方針。

## 3. ビジュアル・アイデンティティ

### 3-1. カラーパレット (実コードから抜粋)

```
■ キャンバス (背景)
  Light Canvas  #F0F0F2   ← タイトル・メニュー (まとも面)
  Gray Canvas   #BBBBBB   ← ステージ選択
  Dark Canvas   #111116   ← 出題・リザルト (戦闘面)

■ ボックス (情報パネル)
  Box Bg        #2A2A2A   ← 暗いボックス (白枠 + 影)
  Box Bg Deep   #151519   ← より沈んだボックス
  Box Border    #FFFFFF   ← 必ず白い枠線で囲う

■ アクセント (差し色 / 必ずこの3色のみ使う)
  Cyan          #00E5FF   ← メイン強調・成功・上位ランク
  Red           #FF3340   ← 失敗・DEAD END・煽り
  Yellow Warn   #FFCC00   ← 警告・OFFLINE・注意
```

**配色ルール**:
- ベース 70%: モノクロ (黒/白/灰)
- 差し色 30%: シアン or レッド (両方を1画面に同時に強く出すと安っぽくなるので片方に寄せる)
- イエローは控えめ (バッジ・警告のみ)

### 3-2. フォント

| 用途 | フォント | 印象 |
|---|---|---|
| 英数字 | **Courier New** (等幅モノスペース) | ターミナル・プログラム感・古いPC感 |
| 日本語 | **Hiragino Kaku Gothic ProN** / Yu Gothic (フォールバック) | 角ゴ・スッキリ |

**重要**: タイトルロゴや見出しは **必ず等幅モノスペース** で組む。「VHS時代の薄暗い情報端末」感が世界観の根幹。

### 3-3. キーモチーフ (世界観の鍵)

#### 1. 🦋 蝶 (Butterfly) — 救済と祝祭のシンボル
- 上位ランク (SS / S / A) クリア時、画面下から蝶が舞い上がる
- ALL CLEAR 達成時にも蝶バースト
- **「狂った世界からの脱出」** のメタファー
- スプライト: `sprite/butterfly.png` 既存 (シルエット型)

#### 2. 📼 VHS / グリッチノイズ
- 画面に走るスキャンライン
- カラー RGB シフト
- 突発的なフレーム消失
- 古いブラウン管・録画劣化テープ感

#### 3. 🔣 浮遊する記号・英数字
- 背景に薄く漂う `#@!?*+=/\\` `0-9` `A-Z`
- 「データの残骸」「壊れたシステムログ」
- 透明度 0.2-0.45、ゆっくり浮遊

#### 4. 💀 DEAD END スタンプ
- 即死エンド時に画面に赤いスタンプが押される
- 文字: `DEAD END` (Courier New, 極太, 赤 #FF3340)
- B級ホラー映画の演出感

#### 5. 👧 ナビゲーター (女の子のキャラクター)
- リザルト画面でランクに応じてセリフを言う
- スプライト: `sprite/girl/` 既存 (複数ポーズ)
- **キャラの存在感は中程度**: メインビジュアルにはしない、隅に居る程度
- 必要ならスクリーンショットの隅に登場させてOK

### 3-4. UI のテクスチャ感

- **二重白枠** `border: 4px double #FFF` 多用
- **オフセット影** `box-shadow: 8px 8px 0 #000` (黒の角ばった影、ピクセル感)
- 角丸は使わない (基本シャープな矩形)
- グラデーションは控えめ (使ってもモノトーン)

## 4. トーン & コピー (文字情報の口調)

### 4-1. ゲーム内の文体 (これ重要)
- **煽り口調** だが悪意はない、むしろ友達の冗談に近い
- 子供っぽさと辛辣さが同居
- 死語や流行語を絶妙に混ぜる

### 4-2. 上位 % ラベル (達成感バズ)
クリアランクに応じて表示される煽りラベル:

| ランク | ラベル例 |
|---|---|
| SS (0.01%) | 宇宙飛行士選抜通過 / ノーベル賞候補 / プロ棋士レベル |
| S (1%) | 医師 / 年収2000万円 / 東大生 |
| A (3-5%) | 早慶合格率 / TOEIC 900超 |
| B (10%) | 公務員合格 / 大卒上位 |

### 4-3. 下位ラベル (ネタバズ用、辛辣)
低ランクの方こそ **スクショ拡散される設計**:

| ランク | ラベル例 |
|---|---|
| C-D (普通) | 凡人 / 人類の中央値 |
| E (下位30%) | 小学生に負けるレベル |
| F (下位5%) | **「現代の縄文人」** / **「令和の維管束知らず」** / 伝説のアホ |

### 4-4. ストア用コピー (素材)
スクリーンショット内のキャッチコピー候補:

- 「あなたは維管束を覚えていますか?」
- 「画面が壊れたまま、小6クイズに挑戦しろ。」
- 「正解を選んだはずなのに、なぜか不正解。」
- 「DEAD END まで、あと 3 問。」
- 「これは脳が腐っているのか、UI が壊れているのか。」
- 「全 1000 問。あなたが忘れた知識のすべて。」
- 「Sランク達成率 0.01%」
- 「現代の縄文人、あなたは仲間を見つけた。」
- 「小学生のとき、できたはずなのに。」

## 5. ゲーム画面 (スクショ素材として強い場面)

実際にスクショを撮るときに **絵的に強い瞬間** はこれ:

### A. タイトル画面
- 黒背景に白い「変なクイズ」ロゴ + サブタイトル "ODD QUIZ"
- "TAP TO START" の T の字に蝶が止まっている (実装済み)
- 背景に薄く浮遊する英数字記号
- フッターに `v0.9.0 / プレイヤー名`

### B. ステージ選択 (整然)
- グレー背景 + 1〜10 ステージのカード群
- 解放済みは白枠、未解放はロックアイコン
- まとも UI を見せる「だましの画面」

### C. 出題画面 (崩壊状態) ★ 一番映える
- 黒背景に四角い問題ボックス (白二重枠)
- 画面が反転 / 回転 / グリッチしている
- 選択肢ボタンが揺れる・色が反転する
- **崩壊度別に複数枚撮ってバリエーション**

### D. リザルト画面 (S ランク)
- 黒背景にでかい `S` ランク文字 (シアン+赤の二重影)
- 蝶が画面いっぱいに舞っている
- 「上位 0.01% / ≒ 早慶合格率と同等」
- スコア・タイム・正解数

### E. リザルト画面 (DEAD END)
- 真っ赤な「DEAD END」見出し
- ランク `F` の赤い大文字
- 「現代の縄文人」「令和の維管束知らず」のラベル
- 画面全体に微細ノイズが乗っている
- ナビゲーター女の子が左下で慰めている

### F. ランキング画面
- 黒背景に TOP100 のリスト
- 1〜3 位はメダル色 (金/銀/銅) で強調
- 自分のエントリだけシアンの枠線で光ってる
- ヘッダ右に `● ONLINE` シアンバッジ

### G. シェア画像 (ゲーム内生成のスクショ)
- 既にゲーム内で `html2canvas` 経由で生成される
- これそのものを App Store のスクショに使ってもいい

## 6. App Store スクリーンショット要件

> **詳細な技術仕様は [`APPSTORE_SPEC.md`](./APPSTORE_SPEC.md) を参照**。  
> ここではデザイン制作に必要な要点だけ抜粋する。

### 6-1. 必須サイズ (2026年4月時点)

| デバイス | 解像度 (Portrait) | 枚数 |
|---|---|---|
| **iPhone 6.9"** (iPhone 16/17 Pro Max) | ⭐ **1320 × 2868** (縦) | 必須 / 推奨 8 枚 (最大10) |
| iPhone 6.5" (iPhone 11/XS Max) | 1284 × 2778 (縦) | 任意 (6.9 から自動縮小) |
| iPad 13" | 2064 × 2752 (縦) | iPad対応する場合のみ ← **今回は不要** |

> **重要**: Apple は 2024 年から **iPhone 6.9" のみ必須** に変更。**1320×2868 を 8 枚** 作れば最低要件 OK。他デバイスは自動スケーリング。

### 6-2. 推奨スクショ構成 (8 枚)

| # | 主題 | レイアウト案 |
|---|---|---|
| 1 | **コンセプトの一発目** | 黒背景に大きく「変なクイズ / ODD QUIZ」+ 一言コピー「画面が壊れたまま、小6クイズに挑戦しろ。」 |
| 2 | **崩壊の凄み** | 出題画面のスクショ (Stage 8 くらい、複数崩壊同時発動) + コピー「20種類の崩壊UIが襲ってくる。」 |
| 3 | **共感バズ** | 「維管束」「班田収授法」を含む実際の問題画面 + コピー「あなたは維管束を覚えていますか?」 |
| 4 | **S ランク達成感** | リザルトS画面 + 蝶 + コピー「Sランク達成率 0.01% — 神になれ。」 |
| 5 | **ネタ枠 (下位)** | リザルトF画面 + DEAD END + 「現代の縄文人」 |
| 6 | **ランキング画面** | TOP100 が見える画面 + コピー「全国のクソゲー人と競え。」 |
| 7 | **ゲームの幅** | 6ジャンル (算数/英語/国語/理科/社会/家庭科音楽美術体育) のアイコン羅列 + 「全1000問」 |
| 8 | **行動喚起** | 「あなたは何ランク? — 今すぐダウンロード」 + ロゴ |

### 6-3. デザイン制約 (Apple ガイドライン)

- ❌ 「Free」「No.1」「Best」など誇大表現は審査で弾かれる可能性あり
- ❌ 他社アプリのアイコンや UI は使わない
- ❌ Apple のロゴ・iPhone のシルエットを画像に入れない
- ❌ "Top of the Charts" 系のランキング自称はダメ
- ✅ 自分のアプリの UI を加工するのは OK
- ✅ コピーで世界観を語るのは推奨

### 6-4. アイコン (1024 × 1024)
別途 App Store 用の 1024×1024 アイコンが必要。  
既存の `apple-touch-icon.png` (180×180) のデザインを **1024px に再描画** する必要がある。  
拡大してもボケない vector / 高解像度ソースで作り直すのが望ましい。

## 7. デザインの方向性チートシート

### こうしてほしい (Yes)
- 黒背景 + シアン or 赤の差し色 + 白い等幅文字
- 古いブラウン管 / VHS / 80年代PCターミナル感
- 蝶モチーフ (上位ランクや祝祭演出に)
- グリッチ / RGB シフト / スキャンライン
- 角ばった矩形・二重白枠・オフセット影
- 「煽ってくる文字情報」(辛辣だが笑える)

### こうしないでほしい (No)
- ❌ パステルカラー / 可愛い系 / 丸っこい UI
- ❌ 写真リアル風 / フォトリアリスティック
- ❌ ゲームっぽい派手なロゴ装飾 (RPG風グラデやキラキラ)
- ❌ 子供向けっぽい雰囲気 (一見クイズアプリだが大人ターゲット)
- ❌ シリアスすぎ / 説教くさい
- ❌ 白背景メインの清潔感ある UI (それは "ふつうのクイズアプリ" になる)
