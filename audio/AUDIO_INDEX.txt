【参照する実装側の主要ポイント】
  - 既存の audio 再生フック: 後述の実装箇所セクション参照
  - 画面状態: body[data-screen="title|stageSelect|question|result"]
  - ギミック状態: engine が body 等に is-active class を付与
  - 呼び出し例:
      Audio.play('se/system/correct.mp3')
      Audio.playLoop('se/gimmick/glitch_loop.mp3')
      Audio.stop('se/gimmick/glitch_loop.mp3')

【ライセンス】
  全SEは効果音ラボ (soundeffect-lab.info) 由来。商用OK/クレジット不要。
  アプリ内組み込みOK。単体再配布のみ禁止。

  SECTION 6 - 実装側 参考

【推奨 API 形】

  // 単発
  Audio.play(path)                   — overlap 可。同時複数OK
  Audio.playExclusive(path)          — 同キーの既存再生を中断して1つだけ再生
  // ループ
  Audio.playLoop(path, volume=0.3)   — loop=true
  Audio.stop(path)                   — ループ停止
  // 画面遷移
  Audio.abortAll()                   — 画面切替時に全SE停止

【長尺ファイル要注意】

  timeout.mp3      (6.01s)  — 遷移前に abort 必須
  key_ok.mp3       (4.68s)  — 遷移前に abort
  b20_in.mp3       (31.06s) — 先頭3〜5秒のみ使う実装推奨
  glitch_loop.mp3  (17.14s) — loop=true 想定。小音量で持続
  g2_betray.mp3    (4.08s)  — 短い結果表示なら fadeOut
  b16_alarm.mp3    (4.02s)  — 時間切れ瞬間のみ単発

【ボリューム基準 (目安)】

  効果音系 (correct/wrong 等):       1.0   (フル)
  タイプ/カーソル (key_tap/select):  0.6
  ループ系 (glitch_loop):            0.25
  警報 (b16_alarm / time_warn):      0.8
  BGM:                               0.4
