## 1. プロジェクト概要

### 1-1. コンセプト
小学6年生レベルの問題を、崩壊したUIで解かせるスコアアタック型クソゲー。
SNSバズ最大化を目的とした一作目MVP。

### 1-2. 優先順位（変更不可）
**バグなしリリース ＞ 機能性 ＞ こだわり**

### 1-3. ターゲット
iOS / Android（Capacitor 8 による単一コードベース、アプリ化前提）

## 3. ビジュアルデザインシステム

### 3-1. カラーパレット
```
/* ベース（モノクロ） */
--bg-deep:     #050508;  /* 最外コンテナ */
--bg-battle:   #111111;  /* プレイ画面背景 */
--bg-mid:      #2a2a2a;  /* パネル内部 */
--bg-light:    #bbbbbb;  /* ホーム・明るい画面 */
--bg-white:    #f8f8f8;  /* タイトル背景 */

/* テキスト */
--text-main:   #ffffff;
--text-sub:    #888888;

/* アクセント */
--accent-cyan:  #00ffff;  /* メインアクセント、正解・ハイライト */
--accent-blue:  #00aaff;  /* サブ、ボタン */
--accent-mint:  #7dd3c0;  /* ナビゲータ少女の差し色と整合 */

/* 警告 */
--warn-red:    #ff4444;
--warn-soft:   #ff6666;  /* BACKボタンなど */

/* グリッド */
--grid-dark:   rgba(255, 255, 255, 0.06);
--grid-light:  rgba(0, 0, 0, 1);
```

### 3-2. タイポグラフィ
- 基本フォント: `'Courier New', monospace`（3SEC継承）
- 問題文フォント: 同上。崩壊UIで blur / フォントサイズ改変を掛ける対象
- 日本語対応: システムフォント（`'Hiragino Sans', 'Yu Gothic', sans-serif`）にフォールバック

### 3-3. 画面サイズ設計
**仮想解像度 1080 × 1920（縦画面）**
- スマホ縦持ちを前提（クイズは縦UIが自然）
- 3SECは横1920×1080だったがアプリ化に合わせ縦に再設計
- `transform: scale(var(--scale))` で端末に合わせて等倍縮小

### 3-4. 共通UIパーツ
- **3Dボックス**: ダブルボーダー（外枠 #fff、内枠 #888）＋ シャドウ（右下方向、rgba(0,0,0,0.5)）＋ 内背景 #2a2a2a
- **蝶々カーソル**: `sprite/butterfly.png` を左側に浮かべる（選択肢の左端）
- **浮遊文字背景**: タイトル・ホームに英数字記号が ふわふわ floating、透過 25〜40%
- **VHSグリッチ**: スキャンライン・RGB分離・画面揺れ・ブロックノイズ・トラッキングライン
  - タイトル: ランダム発動（5〜15秒間隔）
  - 崩壊UI B07「グリッチ」: 問題画面で 8:2 の頻度で 0.3秒発動

## 5. 画面別仕様

### 5-1. タイトル画面
- 背景: `--bg-white` (#f8f8f8)
- 浮遊文字: 英数字・記号、透過 25〜40%、ふわふわ動く
- ロゴ: 中央大きく表示（タイトル名は未定、仮タイトル「変なクイズ」）
- ナビゲータ画像: 右下 or 左下に `basic.png` 配置
- VHSエフェクト: ランダム発動
- メニュー:
  - START GAME / 記録 / 設定
  - 蝶々カーソルで選択

### 5-2. ステージ選択画面
- 背景: `--bg-light` (#bbb) + 黒グリッド
- レイアウト: 縦スクロール（10ステージ並び）
- 各ステージブロック:
  ```
  ┌────────────────────────────┐
  │ STAGE X                    │
  │ 〜ステージ名〜              │
  │ 崩壊UI: N個 / 推定難度: ★★★ │
  │ ベストランク: A            │
  │ [挑戦する]                  │
  └────────────────────────────┘
  ```
- ロック中のステージは灰色＆鍵アイコン、タップ不可

### 5-3. 問題画面
- 背景: `--bg-battle` (#111) + 白グリッド（透過 6%）
- 上部: 問題番号 / 残り問題 / 簡易タイマー
- 中央: 問題文
- 下部:
  - 選択肢モード: 4択ボタン（2×2）
  - 記述モード: 入力欄 + 内製文字盤 + 確定ボタン
- 崩壊UIの発動要素はこの画面にのみ適用される

### 5-4. リザルト画面（バズ装置）
**親仕様書 8-1 に準拠**。要点再掲：
- 背景: #2a2a2a + 白ダブルボーダー
- ランクを大きく表示（S〜F）
- タイム・スコア・正解数
- 「上位 X%」＋現実換算ラベル（早慶合格率等）
- [シェア] [もう一度] [ホーム] ボタン
- このタイミングでVHSグリッチは使用しない（スクショ品質保持）

### 5-5. 設定画面
- BGM ON/OFF
- SE ON/OFF
- 振動 ON/OFF
- データリセット（確認ダイアログ付き）

### 5-6. 記録画面
- ベストスコア
- クリアステージ数
- 総プレイ回数
- Sランク挑戦回数

## 6. 問題システム

### 6-1. データ構造（既定）
- 場所: `data/questions/{math, english, japanese, science, social, others}.json`
- 全200問、ID体系: `M01..M40 / E01..E40 / J01..J40 / R01..R20 / S01..S20 / O01..O40`
- 詳細は 親仕様書 6章 および各JSONファイル参照

### 6-2. 各問題のフィールド
| フィールド | choice | input | 説明 |
|---|---|---|---|
| `id` | ○ | ○ | ジャンル1文字 + 2桁番号 |
| `genre` | ○ | ○ | "算数" 等 |
| `subgenre` | ○ | ○ | 細分カテゴリ |
| `difficulty` | ○ | ○ | 1〜3 |
| `mode` | ○ | ○ | "choice" / "input" |
| `question` | ○ | ○ | 問題文 |
| `choices` | ○ | - | 4つの選択肢 |
| `answer` | ○ | - | choices内の正解index(0〜3) |
| `answer_text` | - | ○ | ひらがな/英字/数字の正解 |
| `answer_variants` | - | ○ | 別表記の配列 |
| `explanation` | ○ | ○ | 解説文 |

### 6-3. 問題抽選アルゴリズム
- ステージ突入時、ジャンル配分比率（20/20/20/10/10/20%）に従って20問抽選
- ステージ難度に応じてL1/L2/L3の比率を変動（下記ステージ表参照）
- 同一ステージ内では重複なし、異なるステージ間での再出題は許可

### 6-4. 記述モードの判定ロジック
```
入力値を正規化（ひらがな変換、空白除去、大文字→小文字、全角→半角など）
  → answer_text と照合
  → answer_variants に含まれるか照合
  → いずれかに一致すれば正解
```

## 7. 内製文字盤（キーボード）仕様

### 7-1. 方針
iOS/Androidの標準IMEは使わない。**全ての入力を独自実装**する。理由は `ギミック一覧.md` 参照（端末差・IME干渉回避、ズル防止、崩壊UIの自由度確保）。

### 7-2. レイアウト（縦画面下部固定）
```
┌────────────────────────────────────┐
│       入力欄（確定前の文字列）       │
├────────────────────────────────────┤
│ [あ] [か] [さ] [た] [な]           │
│ [は] [ま] [や] [ら] [わ]           │
│ [゛] [小] [←削除] [確定]           │
│ [あア] [ABC] [123] [記号]          │
└────────────────────────────────────┘
```

### 7-3. 入力モード
| モード | 内容 |
|---|---|
| ひらがな（デフォルト） | 10キーフリック、あ行〜わ行 + 特殊 |
| カタカナ | 配置はひらがなと同じ、出力のみカタカナ |
| 英字 | QWERTY配列 or フリック式、小文字/大文字切替 |
| 数字 | 0-9 テンキー |
| 記号 | 一部の問題用、`.`, `,`, `?` など |

### 7-4. 入力方式
- **タップ**: キーを押すと中央の文字を出力
- **フリック**: 押したまま上下左右に動かすと各方向の文字を出力
  - 閾値: 15〜20px
  - 指の移動方向の判定は touchmove の最終位置
- **長押し**: 文字候補ポップアップ（iOS標準と同じ挙動）※α版は省略可

### 7-5. 入力処理
```
keyTap(key) →
  bufferに文字を追加
  inputRender() で入力欄を更新
```
崩壊ギミック（W系）は `keyTap` ハンドラの前後に挿入される。

### 7-6. α版で必須 / 後回しで良い部分
| 項目 | α版 | β以降 |
|---|---|---|
| ひらがな10キー | 必須 | - |
| フリック入力 | 必須 | - |
| 英字モード | 必須 | - |
| 数字モード | 必須 | - |
| カタカナモード | 必須 | - |
| 記号モード | あれば | 強化 |
| 長押し候補ポップアップ | 省略 | 追加 |

## 8. 崩壊UIシステム

### 8-1. システム構造
```
GimmickManager
  ├─ registerGimmick(id, { apply, release, config })
  ├─ applyGimmick(id, params) → 画面に効果付与
  ├─ releaseAll()             → 次問遷移時に全解除
  └─ pickFromStagePool(stage, count) → ステージプールから抽選
```

### 8-2. 1問あたりの発動ギミック数（K）
| Stage | K（基本） | K（最大） | モード制約 |
|---|---|---|---|
| 1 | 0〜1 | 1 | B系のみ |
| 2 | 1 | 1 | B系のみ |
| 3 | 1〜2 | 2 | B系のみ |
| 4 | 2 | 2 | B系 + C系 |
| 5 | 2〜3 | 3 | B系 + C系 |
| 6 | 3 | 3 | B系 + C系 + W系 投入開始 |
| 7 | 3〜4 | 4 | 全系 |
| 8 | 4〜5 | 5 | 全系 |
| 9 | 5〜7 | 7 | 全系、パラメータMAX |
| 10 | 7〜 | 全乗せ | 全系 + 理不尽ギミック |

### 8-3. 抽選ルール
- 各ステージのプールから、問題のmodeに応じて絞り込む
  - `mode: "choice"` → B系 ∪ C系 から
  - `mode: "input"` → B系 ∪ W系 から
- Stage内の20問それぞれで独立に抽選（プレイごとに組合せ変化）
- Stage9: 1ミスごとに次問題の K を +1

## 9. ステージ構成（厳格版）

### 9-1. 設計思想
- **Stage1〜5**: 「おもしろい」範囲。視覚演出系・軽い崩壊のみ。ストレスを与えない
- **Stage6〜7**: マイルドストレス。記述系崩壊を導入開始
- **Stage8〜10**: 超ストレス。全ギミック解放、Stage10は理不尽

### 9-3. ステージ別プール

#### Stage 1 — ウォームアップ
- 想定難度: ★☆☆☆☆
- 問題難度: L1=15, L2=5, L3=0
- プール（3個, Eのみ）: **B08, B16, B18**
- K=0〜1（6割はノーギミック）
- クリア想定: 95%

#### Stage 2 — 軽く崩れる
- 想定難度: ★☆☆☆☆
- 問題難度: L1=10, L2=10, L3=0
- プール（5個, Eのみ）: **B08, B16, B18, B07, B03**
- K=1
- クリア想定: 85%

#### Stage 3 — 見えにくい
- 想定難度: ★★☆☆☆
- 問題難度: L1=5, L2=10, L3=5
- プール（7個, Eのみ）: Stage2 + **B02, B19**
- K=1〜2
- クリア想定: 70%

#### Stage 4 — 崩壊が広がる
- 想定難度: ★★☆☆☆
- 問題難度: L1=0, L2=10, L3=10
- プール（10個, E + Cの軽いもの）: Stage3 + **B11, B15, B17**
  - Cモード問題には **C01 選択肢シャッフル** を追加解放
- K=2
- クリア想定: 50%

#### Stage 5 — 中間関門（おもしろい上限）
- 想定難度: ★★★☆☆
- 問題難度: L1=0, L2=8, L3=12
- プール（12個）: Stage4 + **B04 ズーム暴走, B05 ミラー, B06 色覚破壊**
- K=2〜3
- クリア想定: 30%
- **ここまでが「おもしろい」の範疇**

#### Stage 6 — ストレス入門（W系解放）
- 想定難度: ★★★☆☆
- 問題難度: L3中心（L2=5, L3=15）
- プール（16個）: Stage5 + **B12 ぼかし, B14 余白暴走, B20 暗転, C02 ダミー選択肢**
  - 記述問題には **W01 文字盤見えない, W02 文字盤あべこべ** を追加解放
- K=3
- クリア想定: 15%

#### Stage 7 — かなり難
- 想定難度: ★★★★☆
- 問題難度: L3中心（L3=18, L2=2）
- プール（22個）: Stage6 + **B09 画面縮小, B13 フォント極小, C03 選択肢文字変化**
  - 記述: **W04 入力ズレ, W05 カーソル暴走, W07 入力1文字消失, W18 キー消失**
- K=3〜4
- クリア想定: 7%

#### Stage 8 — 全崩壊発動
- 想定難度: ★★★★☆
- 問題難度: L3 only
- プール（30個）: Stage7 + **B01 反転タップ, B10 問題文ランダム, C04 嘘50:50**
  - 記述: **W03 解答欄見えない, W06 文字順逆転, W09 ゴースト入力, W10 入力遅延, W14 キー巨大化, W17 カタカナ勝手切替, W19 フリック反転, W20 フリック方向シャッフル**
- K=4〜5
- クリア想定: 3%

#### Stage 9 — 最難関
- 想定難度: ★★★★★
- 問題難度: L3 only（選別した20問）
- プール: **Aランク42個全部**
- パラメータMAX:
  - blur: 6px以上
  - 入力遅延: 3秒
  - フェードアウト完了: 3秒
  - キー巨大化: 画面の70%占拠
- **動的切替**: 5秒おきに発動ギミック組み合わせ変動
- **ミスペナルティ**: 1ミスで次問題の K +1
- クリア条件: 20問中15問以上正解
- クリア想定: 1%

#### Stage 10 — ガチキチ（Sランク専用）
- 想定難度: ★★★★★★
- 問題難度: L3 + 特殊問題セット
- プール: Aランク42 + X系 + 理不尽ギミック
- 理不尽ギミック（親仕様書5章 G1〜G8 から）:
  - **G1 ランダム即死**: 問題表示前に10%の確率で失格
  - **G2 誤判定**: 正解しても15%の確率で不正解扱い
  - **G4 文字化け問題**: 50%の問題で文字が記号化
  - **G5 選択肢ワープ**: 送信の瞬間に選択肢位置入替
  - **G7 スコア煽り**: 終盤で「スコア:0」表示→実スコア表示の嘘
  - **G8 易問トラップ**: 「1+1=?」の選択肢が紛らわしい漢字
- X系ギミック: W08, W15, W16, B21 全部投入
- K=全乗せ（20+）
- クリア条件: 20問中20問正解（ノーミス）
- クリア想定: 0.01%

## 10. スコアシステム

### 10-1. スコア計算（親仕様書 8-3 準拠）
```
score = 正解数 × 100
      + タイムボーナス（max 500、早いほど高い）
      + ノーミスボーナス（20連続正解で +500）
      + Stage到達ボーナス（Stage n到達で n×100）
      + 崩壊UI数ボーナス（クリア時の崩壊数×50）
```

### 10-2. ランク判定
親仕様書 8-2 準拠（S/A/B/C/D/E/F）

### 10-3. 上位％ラベル
親仕様書 8-4 準拠。上位・下位ラベルはランダム表示。
