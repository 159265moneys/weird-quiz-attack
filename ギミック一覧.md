# 崩壊UIギミック一覧 v1.0

作成日: 2026-04-21
前提: アプリ内に独自の文字盤(キーボード)を実装する。フリック入力も内製で対応。

---

## 凡例

### 回答方式
- **B** = Both（記述・選択肢のどちらでも発動可）
- **W** = Write専用（記述問題でのみ発動可）
- **C** = Choice専用（選択肢問題でのみ発動可）

### 実装ランク
- **A** = そのまま採用。実装2h前後、バグリスク低
- **B** = 採用だが難あり。工数or状態管理でハマる可能性、α後期〜β向け

### 抽選ルール
- `mode: "choice"` の問題 → B系 + C系 から抽選
- `mode: "input"`  の問題 → B系 + W系 から抽選

## 【A-W】記述専用・そのまま採用（17件）

### 既存ギミック（7件）

| ID | 名前 | 説明 |
|---|---|---|
| W01 | 文字盤見えない | キーボードの文字が非表示、記憶で打つ |
| W02 | 文字盤あべこべ | キー配置がランダム化される（問題ごとに1回固定） |
| W03 | 解答欄見えない | 入力中の文字が見えない、確定するまで分からない |
| W04 | 入力ズレ | 一文字前の音が入力される（「い」→「あ」、「け」→「く」など） |
| W05 | カーソル暴走 | 入力位置が勝手にズレる |
| W06 | 文字順逆転 | 入力した文字が逆順で並ぶ（鏡文字ではない） |
| W07 | 入力1文字消失 | 1文字打つごとに前の文字が非表示に（内部保持、パスワードUI的挙動） |

### 内製文字盤化で昇格（3件）

| ID | 名前 | 説明 |
|---|---|---|
| W08 | 文字盤あべこべv2 | キー配置が1文字打つたびに再ランダム化 |
| W09 | ゴースト入力 | 勝手に文字化け入力が挿入される。削除してから打ち直す必要あり |
| W10 | 入力遅延 | タップから2秒遅れて表示される（内部入力は即時、回答ボタンは有効） |

### 新規追加・キー操作系（5件）

| ID | 名前 | 説明 |
|---|---|---|
| W14 | キー巨大化 | 特定の1キーだけ画面の半分を占拠し、他のキーが埋もれる |
| W15 | キーワープ | タップ成功時、該当キーだけ別の位置にワープする |
| W16 | キー同士くっつく | 押した瞬間、隣のキー2〜3個が同じ文字に変化する |
| W17 | カタカナ/ひらがな勝手切替 | 数秒ごとにキーボードのモードが自動で切り替わる |
| W18 | キー消失 | 一度押したキーが0.5秒後に消える（打鍵履歴で難化） |

### 新規追加・フリック系（2件）

| ID | 名前 | 説明 |
|---|---|---|
| W19 | フリック方向反転 | 上フリックで「お」が出る等、全方向が反転する |
| W20 | フリック方向シャッフル | 上下左右に出る文字が毎回ランダムに変わる |
